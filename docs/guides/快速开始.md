   ```bash
   mvn exec:java -Dexec.mainClass="com.example.rtwarehouse.OdsDataReader"
   ```

3. **Dinky 平台部署**
   - 登录 Dinky 控制台
   - 创建 Flink SQL 作业
   - 导入 `dinky_job_config.json`
   - 提交运行

### 运行前必须完成的配置

⚠️ **重要**: 在运行之前，请务必修改以下配置：

#### 1. 修改 `application.properties`

```properties
# MySQL 配置（必须修改）
mysql.hostname=你的 MySQL 主机地址
mysql.port=端口号
mysql.database=数据库名
mysql.username=用户名
mysql.password=密码

# Kafka 配置（必须修改）
kafka.bootstrap.servers=Kafka 地址
kafka.security.protocol=SASL_SSL 或 PLAINTEXT
kafka.sasl.mechanism=PLAIN
kafka.username=Kafka 用户名
kafka.password=Kafka 密码
```

#### 2. 验证环境

参考 `运行前检查清单.md` 完成所有检查项

## 📈 数据流向

```
MySQL (源表) 
  ↓ CDC
ODS 层 (Kafka)
  ↓ 
DWD 层 (清洗 + 标准化)
  ↓
DIM 层 (维度关联)
  ↓
DWS 层 (轻度汇总)
  ↓
ADS 层 (应用层 - 最终结果)
```

## 🔍 架构概览

### 技术栈
- **计算引擎**: Flink 1.16.3
- **CDC 工具**: Flink CDC 2.4.0
- **消息队列**: Kafka
- **数据存储**: MySQL 8.0+
- **状态后端**: RocksDB
- **部署平台**: Dinky (可选)

### 关键特性
- ✅ 实时数据采集 (MySQL CDC)
- ✅ 流式数据处理
- ✅ Exactly Once 语义保证
- ✅ 分布式状态管理
- ✅ 支持断点续传 (Savepoint/Checkpoint)
- ✅ 异步维度查询

## ⚠️ 注意事项

### 首次运行建议

1. **从小规模开始**
   - 先运行 ODS 层测试
   - 验证数据能正确流入 Kafka
   - 再运行完整流程

2. **监控关键指标**
   - Checkpoint 成功率
   - 端到端延迟
   - Task 反压情况
   - 各层数据量对比

3. **准备好回滚方案**
   - 知道如何停止程序
   - 准备好清理脚本
   - 保存好 savepoint 路径

### 生产环境建议

1. **资源配置**
   - 根据数据量调整 parallelism
   - 配置足够的 Task Manager 内存
   - 设置合理的 Checkpoint 间隔

2. **监控告警**
   - 集成 Prometheus + Grafana
   - 配置异常告警
   - 设置数据质量监控

3. **容灾备份**
   - 启用 Savepoint
   - 配置双机房
   - 定期演练恢复流程

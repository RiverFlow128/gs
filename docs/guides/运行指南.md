# 实时数仓运行指南

## 概述

本实时数仓项目现已完成表结构拆分，支持两种运行方式：
1. **本地 Java 程序运行** - 适合开发测试
2. **Dinky 平台部署** - 适合生产环境

## 前置条件

### 1. 环境要求
- JDK 11+
- Maven 3.6+
- Flink 1.16.3
- MySQL 8.0+
- Kafka 2.8+

### 2. 数据源准备

#### MySQL 数据库
需要准备好以下数据库和表：
- `statics` 数据库（源表）
- `rt_warehouse` 数据库（目标表，用于 DWD/DWS/ADS 层）

#### Kafka 集群
配置阿里云 Kafka 或自建 Kafka 集群

### 3. 配置文件修改

修改 `application.properties` 文件：

```properties
# MySQL 配置
mysql.hostname=你的 MySQL 主机地址
mysql.port=端口号
mysql.database=数据库名
mysql.username=用户名
mysql.password=密码

# Kafka 配置
kafka.bootstrap.servers=Kafka 地址
kafka.security.protocol=安全协议（如 SASL_SSL）
kafka.sasl.mechanism=SASL 机制（如 PLAIN）
kafka.username=Kafka 用户名
kafka.password=Kafka 密码
```

## 方式一：本地 Java 程序运行

### 步骤 1: 编译项目

```bash
cd d:\gz\test-main\test-main
mvn clean package -DskipTests
```

### 步骤 2: 运行主程序

#### 选项 A: 使用 Maven 运行

```bash
mvn exec:java -Dexec.mainClass="com.example.rtwarehouse.RealtimeWarehouseApplication"
```

#### 选项 B: 使用 IDE 运行

在 IntelliJ IDEA 或 Eclipse 中：
1. 找到 `RealtimeWarehouseApplication.java`
2. 右键 -> Run

#### 选项 C: 使用 jar 包运行

```bash
java -jar target/realtime-warehouse-1.0.0.jar
```

## 方式二：Dinky 平台部署

### 步骤 1: 准备 SQL 文件

确保以下文件存在：
- `src/main/resources/sql/ods/tables.sql`
- `src/main/resources/sql/dim/tables.sql`
- `src/main/resources/sql/dwd/tables.sql`
- `src/main/resources/sql/dws/tables.sql`
- `src/main/resources/sql/ads/tables.sql`

### 步骤 2: 上传到 Dinky

1. 登录 Dinky 控制台
2. 创建新的 Flink SQL 作业
3. 作业名称：`实时数仓作业`
4. 作业类型：`Flink SQL`
5. Flink 版本：`1.17.1`

### 步骤 3: 配置作业参数

并行度：`2`（根据实际情况调整）

Checkpoint 配置：
- 启用：是
- 间隔：`30000ms`
- 模式：`EXACTLY_ONCE`

Savepoint 配置：
- 启用：是
- 路径：`hdfs://your-hdfs-endpoint:9000/savepoints`

State Backend：
- 类型：`ROCKSDB`
- Checkpoint 路径：`hdfs://your-hdfs-endpoint:9000/checkpoints`

### 步骤 4: 提交作业

将 `dinky_job_config.json` 导入或直接配置后提交作业。

## 验证数据流

### 1. 检查 ODS 层

查看 Kafka Topic 是否有数据：

```bash
# 列出所有 topic
kafka-topics.sh --bootstrap-server <kafka-server> --list

# 消费特定 topic 查看数据
kafka-console-consumer.sh --bootstrap-server <kafka-server> \
  --topic game_record_ods \
  --from-beginning
```

### 2. 检查 DWD 层

查询 MySQL 中的 DWD 表：

```sql
SELECT * FROM dwd_game_record LIMIT 10;
```

### 3. 检查 DWS 层

查询 MySQL 中的 DWS 汇总表：

```sql
SELECT * FROM dws_game_record_daily LIMIT 10;
```

### 4. 检查 ADS 层

查询最终的结果表：

```sql
SELECT * FROM ads_today_user_game_statistic LIMIT 10;
```

## 常见问题排查

### 问题 1: 无法连接 MySQL CDC

**症状**: 报错 "Unable to connect to MySQL"

**解决方案**:
1. 检查 MySQL 是否开启 binlog
2. 检查网络连接
3. 确认账号权限

```sql
-- 检查 binlog 状态
SHOW VARIABLES LIKE 'log_bin';
SHOW VARIABLES LIKE 'binlog_format';
```

### 问题 2: Kafka 连接失败

**症状**: 报错 "Failed to connect to Kafka"

**解决方案**:
1. 检查 Kafka 服务是否正常
2. 确认 security.protocol 配置正确
3. 检查 SASL 认证信息

### 问题 3: Checkpoint 失败

**症状**: Checkpoint 超时或失败

**解决方案**:
1. 增加 checkpoint 间隔时间
2. 优化状态后端配置
3. 检查 HDFS 存储空间

### 问题 4: 数据延迟

**症状**: 数据写入延迟较高

**解决方案**:
1. 增加 parallelism
2. 优化 SQL 逻辑
3. 检查资源使用情况

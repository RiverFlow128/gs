### 2️⃣ ClickHouse JDBC 依赖冲突 ✓ 已修复

**问题描述**：
```json
// dinky_job_config.json - 两个冲突的依赖
{
  "groupId": "com.clickhouse",
  "artifactId": "clickhouse-jdbc",
  "version": "0.3.2-patch11"
},
{
  "groupId": "ru.yandex.clickhouse", 
  "artifactId": "clickhouse-jdbc",
  "version": "0.3.2"
}
```

**风险**：类加载冲突，可能报错 `ClassNotFound` 或 `MethodNotFound`

**修复方案**：
```json
// 只保留一个新版驱动
{
  "groupId": "com.clickhouse",
  "artifactId": "clickhouse-jdbc",
  "version": "0.4.6"  // 更新版本，修复更多 bug
}
```

**当前状态**：✅ 已删除旧版 `ru.yandex.clickhouse`，升级到 `0.4.6`

---

### 3️⃣ 编码问题根治 ✓ 已修复

**问题描述**：
- 依赖 `fix_encoding.py` 脚本硬修乱码
- Git 未强制 UTF-8 编码
- IDE 编码设置不一致

**风险**：每次新人拉代码、IDE 格式化都可能产生新乱码

**修复方案**：

#### 1. 创建 `.gitattributes` 文件
```gitattributes
*.sql text eol=lf working-tree-encoding=UTF-8
*.java text eol=lf working-tree-encoding=UTF-8
*.properties text eol=lf working-tree-encoding=UTF-8
*.xml text eol=lf working-tree-encoding=UTF-8
*.json text eol=lf working-tree-encoding=UTF-8
*.md text eol=lf working-tree-encoding=UTF-8
```

#### 2. Maven 配置（已完成）
```xml
<pom.xml>
<project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
```

#### 3. IDEA 设置指南
创建了 [`IDE 编码设置指南.md`](./IDE 编码设置指南.md)，请按步骤设置

**当前状态**：✅ 从源头杜绝编码问题，可以删除 `fix_encoding.py`

## ✅ 已完成的优化

### 4️⃣ HDFS 路径本地化 ✓ 已优化

**原配置**：
```json
"savepoint": {
  "path": "hdfs://your-hdfs-endpoint:9000/savepoints"
}
```

**问题**：本地测试没有 HDFS，启动失败

**修复方案**：
```json
"savepoint": {
  "path": "file:///tmp/savepoints"  // 本地文件系统
},
"stateBackend": {
  "checkpointPath": "file:///tmp/checkpoints"
}
```

**说明**：
- ✅ Windows 自动映射为 `C:/tmp/savepoints`
- ✅ Linux/Mac 使用 `/tmp/savepoints`
- ✅ 生产部署时再改为 HDFS 路径

### 5️⃣ 添加 Flink Shade 插件 ✓ 已添加

**新增配置**：
```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-shade-plugin</artifactId>
    <version>3.4.1</version>
    <configuration>
        <createDependencyReducedPom>false</createDependencyReducedPom>
        <filters>
            <filter>
                <artifact>*:*</artifact>
                <excludes>
                    <exclude>META-INF/*.SF</exclude>
                    <exclude>META-INF/*.DSA</exclude>
                    <exclude>META-INF/*.RSA</exclude>
                </excludes>
            </filter>
        </filters>
        <transformers>
            <transformer implementation="org.apache.maven.plugins.shade.resource.ServicesResourceTransformer"/>
        </transformers>
    </configuration>
</plugin>
```

**作用**：
- ✅ 打包所有依赖到 fat jar
- ✅ 提交到 YARN/Dinky 时不会缺类
- ✅ 排除签名文件，避免安全异常
